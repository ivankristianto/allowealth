# Demo Mode Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `DEMO_MODE` environment flag that shows a persistent warning banner on every page, hides account management UI, and supports daily data reset via `aw demo reset` CLI + GitHub Actions.

**Architecture:** A single `isDemoMode()` utility reads `getEnv('DEMO_MODE')` at SSR render time (never at module load). The `DemoBanner` component renders in `MainLayout` above the Header. Each restricted page calls `isDemoMode()` in its frontmatter and conditionally omits restricted sections from the DOM. A new `aw demo` CLI command chains `db empty` + `db seed` for both SQLite and D1; a GitHub Actions workflow calls it on a daily cron.

**Tech Stack:** Astro 5 SSR, DaisyUI v5, Cloudflare Workers, `getEnv`/`setTestEnv` from `@/lib/env`, `exec` helper from `src/cli/lib/exec.ts`, `bun:test`, citty CLI, GitHub Actions

import { getEnv } from '@/lib/env';

/**
 * Returns true when the app is running in demo mode.
 *
 * IMPORTANT: Call only at SSR request time (inside Astro frontmatter or API
 * handlers), never at module load time. On Cloudflare Workers, runtime env is

  return getEnv('DEMO_MODE') === 'true';

- [ ] **Step 7.1: Create the demo command**

`exec` is a top-level static import — same as `db.ts` line 15. `resolveTarget` is lazy-imported inside `run()` because it triggers `.env.production` loading as a side-effect for D1 targets.

Add a `--yes` flag: when omitted on a `d1` target, prompt for confirmation (same pattern as `aw db drop`). CI passes `--yes` to skip the prompt.

```ts
// src/cli/commands/demo.ts
/* eslint-disable no-console -- CLI output is intentional */
import { defineCommand } from 'citty';
import { exec } from '../lib/exec';
import { targetArg } from '../lib/target';

export default defineCommand({
  meta: { name: 'demo', description: 'Demo environment management commands' },
  subCommands: {
    reset: defineCommand({
      meta: {
        name: 'reset',
        description: 'Wipe all data and reseed with demo dataset (empty + seed --months=3)',
      },
      args: {
        target: targetArg,
        yes: {
          type: 'boolean',
          description: 'Skip confirmation prompt (required for non-interactive CI use)',
          default: false,
          alias: 'y',
        },
      },
      async run({ args }) {
        const { resolveTarget, getTarget } = await import('../lib/target');
        await resolveTarget(args);

        const target = getTarget();

        if ((target === 'd1' || target === 'd1-local') && !args.yes) {
          console.error('⚠️  This will wipe ALL data on D1 and reseed with demo data.');
          console.error('   Target:', target);
          console.error('\nType "yes" to confirm:');

          const readline = await import('readline');
          const rl = readline.createInterface({ input: process.stdin, output: process.stderr });
          const answer = await new Promise<string>((resolve) => {
            rl.question('', (a) => {
              rl.close();
              resolve(a.trim());
            });
          });

          if (answer !== 'yes') {
            console.error('Aborted.');
            process.exit(1);
          }
        }

        console.log('Resetting demo data...');
        // exec uses execFileSync with argv array — no shell injection risk
        exec('bun', ['run', 'src/db/empty.ts']);
        exec('bun', ['run', 'src/db/seed/index.ts', '--months=3']);
        console.log('✅ Demo data reset complete.');
      },
    }),
  },
});
```

### Task 8: GitHub Actions workflow — daily demo reset

**Files:**

- [ ] **Step 8.1: Create the workflow file**

```yaml
# .github/workflows/demo-reset.yml
name: Demo Reset

on:
  schedule:
    # Runs daily at 00:00 UTC. Adjust cron expression to change timing.
    - cron: '0 0 * * *'
  workflow_dispatch:
    # Allows manual trigger from the GitHub Actions UI

jobs:
  reset:
    name: Reset demo data
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Reset demo data on D1
        # AW_TARGET=d1 + D1_ENABLED=true replaces --target d1 to avoid loadEnvFile('.env.production')
        # which is not present in CI. Wrangler picks up CLOUDFLARE_API_TOKEN automatically.
        # --yes skips the interactive confirmation prompt.
        run: bun run aw demo reset --yes
        env:
          AW_TARGET: d1
          D1_ENABLED: 'true'
          CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
```
