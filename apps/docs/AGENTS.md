# Allowealth Docs - Agent Instructions

## Overview

This is the documentation site for Allowealth, deployed to Cloudflare Pages at `docs.allowealth.io`. It uses Astro with the Starlight documentation theme.

## Tech Stack

| Category      | Technology                     |
| ------------- | ------------------------------ |
| Framework     | Astro 5.x (static output)      |
| Theme         | @astrojs/starlight             |
| Styling       | Custom CSS (Starlight + brand) |
| Sitemap       | @astrojs/sitemap               |
| Deployment    | Cloudflare Pages               |
| Image Hosting | Cloudflare R2                  |

## Adding New Documentation

### 1. Create the file

Add a new `.md` or `.mdx` file in the appropriate `src/content/docs/` subdirectory:

```markdown
---
title: Page Title
description: Brief description for SEO.
sidebar:
  label: Sidebar Label
  order: 3
---

# Page Title

Content here...
```

### 2. Update the sidebar

If the page needs to appear in the sidebar, edit `astro.config.mjs`:

```javascript
sidebar: [
  {
    label: 'Guides: End Users',
    items: [
      // Add your new page here
      { slug: 'end-users/your-new-page' },
    ],
  },
];
```

Or use autogenerated directories:

```javascript
{
  label: 'Reference',
  autogenerate: { directory: 'reference' },
}
```

### 3. Add frontmatter

Required frontmatter fields:

| Field         | Required | Description                              |
| ------------- | -------- | ---------------------------------------- |
| `title`       | Yes      | Page title (appears in browser tab, SEO) |
| `description` | No       | Meta description for SEO                 |
| `sidebar`     | No       | Sidebar label and order                  |
| `draft`       | No       | Set to `true` to hide from production    |

## Screenshots and Images

### Image Hosting

All screenshots **must** be hosted on Cloudflare R2 bucket (`allowealth-docs-screenshots`).

**Base URL:** `https://images.allowealth.io/`

### Uploading Images

```bash
# Upload to R2
bunx wrangler r2 object put "allowealth-docs-screenshots/<filename>" \
  --file "<path-to-local-file>" \
  --remote
```

### Referencing Images

```markdown
![Description](https://images.allowealth.io/filename.png)
```

**Do not** commit screenshots to git. Delete local files after uploading.

## Deployment to Cloudflare Pages

### Manual Deployment

```bash
# Build the site
bun run build

# Deploy using wrangler
bunx wrangler pages deploy
```

### Automatic Deployment (GitHub Actions)

The docs deploy automatically on pushes to `main` that affect `apps/docs/**`:

- Workflow: `.github/workflows/deploy-docs-site.yml`
- Trigger: Push to `main` branch with changes in `apps/docs/**`

### First-Time Setup (Cloudflare Pages)

1. **Create Pages project** in Cloudflare Dashboard:
   - Project name: `allowealth-docs`
   - Production branch: `main`

2. **Configure build settings**:
   - Build command: `bun run build`
   - Build output directory: `dist`
   - Root directory: `apps/docs`

3. **Add custom domain**:
   - Add `docs.allowealth.io` as a custom domain
   - Configure DNS to point to Cloudflare Pages

4. **Create R2 bucket for screenshots**:
   ```bash
   wrangler r2 bucket create allowealth-docs-screenshots
   ```

## Starlight Customization

### Custom CSS

Add custom styles in `src/styles/brand.css`:

```css
/* Custom brand colors */
:root {
  --sl-color-accent: #10b981;
}
```

### Components

Starlight provides built-in components:

- `<Card>` - Info cards
- `<CardGrid>` - Grid of cards
- `<LinkCard>` - Card with link
- `<Tabs>` / `<TabItem>` - Tabbed content
- `<Steps>` - Numbered step lists
- `<FileTree>` - Directory trees
- `<Badge>` - Status badges

Example:

```mdx
import { Card, CardGrid } from '@astrojs/starlight/components';

<CardGrid>
  <Card title="Getting Started" icon="rocket">
    Install and run locally.
  </Card>
  <Card title="Deployment" icon="seti:terraform">
    Deploy to production.
  </Card>
</CardGrid>
```

## Writing Guidelines

1. **Audience-first**: Write for the specific audience (end-user, admin, or developer)
2. **Action-oriented**: Start with verbs ("Click", "Run", "Configure")
3. **Code examples**: Include copy-paste ready commands
4. **Screenshots**: Use sparingly, update when UI changes
5. **Cross-links**: Link to related pages using relative paths
